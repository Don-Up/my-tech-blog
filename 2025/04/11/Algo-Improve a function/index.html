The goal of this problem is to efficiently filter out objects from an array (items) based on a set of exclusion conditions (excludes). Each object in items has at least three properties, and excludes is an array of { k, v } pairs that define which key-value pairs should be removed from items.

Implementation Code

Step-by-Step Explanation

1. Understanding the Problem

  • items is an array of objects, each containing at least three properties.
  • excludes is an array of objects where each object has:
    • k: the key (property) to check in items
    • v: the value that should be excluded
  • The function should return a new array that excludes any object from items that matches any condition in excludes.

2. Optimizing the Exclusion Lookup

  • Instead of scanning excludes repeatedly for every item, we preprocess excludes into a Map for O(1) lookup time.
  • The Map stores:
    • Keys: Property names (k).
    • Values: A Set of values to exclude for that key.

This allows us to check if a value should be excluded in constant time O(1) instead of iterating over excludes (which would be O(m) per item).

3. Filtering the Items Efficiently

  • We iterate through items and check if any of its key-value pairs match an exclusion rule.
  • If any key-value pair in the item is found in excludeMap, the item is removed.

  • .every(...) ensures that an item is only kept if none of its key-value pairs match an exclusion rule.

4. Complexity Analysis

Step Operation Complexity
Preprocessing excludes Creating excludeMap O(m)
Filtering items Checking each key in excludeMap (O(1) lookup) O(n * k)
Overall Complexity O(m) + O(n * k) → O(n * k + m)

Where:

  • n = number of items
  • m = number of excludes
  • k = average number of keys per object in items

This is much more efficient than a naive approach that would require O(n * m) checks.

Key Takeaways

  • Optimized Lookup:
    • Using Map<key, Set<values>> ensures O(1) lookup time instead of iterating over excludes multiple times.
  • Efficient Filtering:
    • Iterating over items once (O(n)) and checking each key (O(k)) makes the solution O(n * k + m).
  • Scalability:
    • This approach works efficiently even for large datasets.

This is a well-optimized solution that balances readability and performance, making it a great approach for filtering objects based on exclusion rules. 🚀
